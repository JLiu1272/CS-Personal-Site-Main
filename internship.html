Octopus Limited Hong Kong - 2018 Summer

Position: Embedded Software Engineer Intern

I had the opportunity to work for Octopus Limited last summer. I was placed on their front-end team, which was the team that maintains and makes the hardware and software for RFID/NFC payment transaction on various vendor locations. The vendor locations include Subways, boutique shops, 7-11 etc...

Project 1: "全城Wi-Fi" (Translated: Wi-Fi All Across Hong Kong")
  • Project Proposal: Allow citizens to use Wi-Fi on the bus. This is done by enabling a dual function on the payment hardware (which is available on all buses in Hong Kong). The hardware not only is capable of receiving payments, but it should also be able to handle Wi-Fi signals
  • I worked on writing scripts to stress test the Wi-Fi signal. The stress test criteria included; Wi-Fi works for prolong period, Wi-Fi speed is adequate even when multiple devices are using the Wi-Fi simultaneously.
  • A script was written to simulate the worse possible environment that the Wi-Fi device may encounter. The script then prints out a report determining whether the test passed the stress test or not. If it fails, it indicates where specifically did it fail
  • Technologies Used: Bash, Busybox (The swiss-knife of embedded systems)
Project 2: Octopus I-Pad Integration
  • Worked on the client-server architecture, which allows an I-Pad to communicate with the hardware
  • This application was developed using Java, TCP/IP Protocol (aka socket) to communicate with hardware, and threading concepts to receive real time messages sent from card reader to be displayed on android application.
Project 3: Made updates to their internal software written in C. The internal software is used to show byte allocations on the RFID card.
